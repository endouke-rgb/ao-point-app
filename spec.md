## 1. アプリ概要

| 項目 | 内容 |
|------|------|
| アプリ名 | アオのポイント管理アプリ |
| 対象ユーザー | アオ・保護者 |
| 形式 | シングルHTMLファイル（ブラウザで動作） |
| 保存先 | **Firebase Realtime Database（クラウド）＋ localStorage（オフライン補完）** |
| 目的 | 公文学習の習慣化をポイント制とシール帳で楽しく後押しする |

---

## 2. ユーザー・モード

**モード分けなし。** 全員が同じ画面でポイントの登録・確認・消費を行う。

---

## 3. ポイント獲得ルール

### 3-1. 公文の宿題（前日までに完了）

| 科目 | ポイント | 備考 |
|------|----------|------|
| 算数 | 1pt | |
| 国語 | 1pt | |
| 英語 | 1pt | |

- 同日に複数科目の登録可（最大 3pt）
- 授業前日までの日付で登録する

### 3-2. 授業態度（やりたくないと言わずに集中して受ける）

| 条件 | ポイント | 備考 |
|------|----------|------|
| 授業日に達成 | 1pt | 授業日（月・木）のみ登録可 |

### 3-3. 授業内テスト 100点

| 科目 | ポイント | 備考 |
|------|----------|------|
| 算数 | 1pt | |
| 国語 | 1pt | |
| 英語 | 1pt | |

- 授業日（月・木）のみ登録可
- 同日に複数科目可（最大 3pt）

**1日の最大獲得ポイント**
- 宿題登録日（非授業日）：最大 3pt
- 授業日（月・木）：最大 7pt（宿題3 + 態度1 + テスト3）

## 4. 画面・機能一覧

### 4-1. ホーム画面（カレンダー）

- 月単位のカレンダーを表示
- 月の移動ボタン（前月・翌月）
- 授業日（月・木）をカラーでハイライト
- ポイントが記録された日にバッジ（獲得pt数）を表示
- 日付をタップ → ポイント登録パネル（モーダル）を開く
- 画面上部に累積ポイントを常時表示

### 4-2. ポイント登録パネル（モーダル）

```
選択中: ○月○日（曜日）

【宿題（前日までに完了）】
  □ 算数  +1pt
  □ 国語  +1pt
  □ 英語  +1pt

【授業態度】※授業日のみ表示
  □ 集中してやりたくないと言わずに受けた  +1pt

【テスト100点】※授業日のみ表示
  □ 算数  +1pt
  □ 国語  +1pt
  □ 英語  +1pt

獲得予定: ○pt
[登録する]ボタン
```

- 授業日（月・木）以外では「授業態度」「テスト100点」セクションを非表示
- 登録済みの日を再度開いた場合、既存の状態を表示（上書き保存）
- 二重加算なし（上書き方式）
- 保存時にシール獲得条件を判定し、条件を満たした場合はシール獲得フローへ遷移

### 4-3. ポイント消費画面

```
現在のポイント: ○○pt

消費するポイント数: [____]
メモ（何に使ったか）: [________]

[ポイントを使う]ボタン
```

- 残高不足の場合はエラーメッセージを表示
- 消費履歴に記録（日付・消費pt・メモ）

### 4-4. シール帳画面 ★新規追加

- 獲得したシールを一覧表示（4列グリッド、紙のテクスチャ背景）
- 各シールに絵文字・名前・獲得日・獲得条件を表示
- シールは新しい順に並ぶ
- シールがない場合は「まだシールがないよ」メッセージを表示

### 4-5. 履歴画面

- ポイント獲得履歴（日付・項目・獲得pt）
- ポイント消費履歴（日付・消費pt・メモ）
- 累積獲得 / 累積消費 / 残高 のサマリー
- 各レコードに「削除」ボタンを追加
  - 削除前に確認ダイアログを表示（「この記録を削除しますか？」）
  - 獲得記録を削除した場合 → カレンダーのバッジも同時に消える
  - 消費記録を削除した場合 → 残高が即時回復する
  - 削除後はヘッダーの残高ポイントをリアルタイム更新

## 5. シール帳機能 ★新規追加

### 5-1. シール獲得条件

| 条件 | トリガー |
|------|---------|
| 宿題コンプリート | 1回の登録で算数・国語・英語の3科目すべての宿題にチェックが入った時 |
| テスト100点 | 授業日（月・木）に1科目以上のテスト100点にチェックが入った時 |

- 再編集時は前回未達成→今回達成の場合のみシールを付与（二重付与なし）
- 1回の保存で両条件を同時に満たした場合は順番にシール獲得フローを2回実行

### 5-2. シール獲得フロー

```
条件達成
  ↓
派手な演出（全画面お祝いアニメーション）
  ↓
シール選択モーダル（タブ切り替え）
  ├─ [絵文字シール] タブ
  │    検索ワード入力 → 候補5つ表示（絵文字 + 名前 + グラデーション背景）
  │    1つを選択 → [このシールにする]ボタン
  └─ [写真シール] タブ
       写真アップロード / カメラ撮影 → プレビュー表示 → シール枠選択（丸・角丸・ハート型）
       → [このシールにする]ボタン
  ↓
Firebase に保存 → シール帳画面へ遷移
```

- スキップ可能（「あとで選ぶ」ボタン）

### 5-3. お祝いアニメーション

- 全画面ダークオーバーレイ（z-index: 300）
- 放射状グラデーション回転アニメーション
- 絵文字がランダムに降るエモジレイン演出
- バイブレーション（Web Vibration API）

### 5-4. シールデザイン

- 絵文字ベースのシール（外部API不要）
- 5種のビジュアルテンプレート（円形・角丸正方形、グラデーション色違い）
- 約75種のシールデータベース（キーワード検索対応）
- 検索ワード例：「動物」「食べ物」「宇宙」「お姫様」など

### 5-5. 写真シール機能 ★新規追加

- シール選択モーダルに「写真シール」タブを追加
- **写真入力方法**
  - ファイル選択（端末のカメラロール・ファイルから選択）
  - カメラ直接撮影（`capture="environment"` または `capture="user"` 対応端末のみ）
- **シール枠（フレーム）**：3種類から選択
  - 丸型（circle）
  - 角丸正方形（rounded）
  - ハート型（heart） ※SVG clip-path で実装
- **プレビュー**: 選択した写真 + フレームをリアルタイムでCanvas描画してプレビュー表示
- **保存形式**: Canvas を `toDataURL('image/jpeg', 0.6)` で Base64 化し Firebase Realtime Database に保存
  - 1枚あたり概算サイズ: 58×58px JPEG ≒ 2〜5KB（Firebase 無料枠内で十分）
- **シール帳表示**: `emoji` フィールドが空で `photoData` フィールドがある場合は `<img>` タグで表示、フレーム形状を CSS で適用
- **バリデーション**
  - 対応形式: image/*（JPEG/PNG/GIF/WEBP）
  - ファイルサイズ上限: 10MB（超えた場合はエラーメッセージ）

## 6. 画面遷移

```
ホーム（カレンダー）
  ├─ 日付タップ        → ポイント登録パネル（モーダル）
  │                        └─ 保存時に条件達成 → お祝い演出 → シール検索 → シール帳
  ├─ 「使う」ボタン    → ポイント消費画面
  ├─ 「シール帳」ボタン → シール帳画面
  └─ 「履歴」ボタン    → 履歴画面
                              └─ 削除ボタン → 確認ダイアログ → 削除実行
```

## 7. データ設計

### 7-1. 保存先

- **Firebase Realtime Database** を使用（無料枠：1GB保存 / 10GB転送／月）
- データはリアルタイムに全端末へ自動反映される
- オフライン時はFirebase SDKがローカルキャッシュで補完し、復帰後に自動同期

### 7-2. 同期の仕組み

```
親のスマホ         Firebase              子のタブレット
   │   ─── 書き込み ──→   DB    ─── リアルタイム通知 ──→  │
   │   ←── 更新受信 ───   DB    ←─────────────────────  │
```

- 同一の「ファミリーコード（任意の文字列）」を両端末で入力することで同じデータを共有する
- ファミリーコードは初回起動時に設定画面で入力

### 7-3. データ構造（Firebase パス）

```
/families/{familyCode}/
  records/
    2025-06-16/
      items/
        hw_math: true
        hw_japanese: true
        ...
      points: 4
  consumptions/
    1718500000000/
      date: "2025-06-16"
      points: 10
      memo: "ゲームの時間30分"
  stickers/
    1718500000000/
      date: "2025-06-16"
      emoji: "🦁"          # 絵文字シールの場合。写真シールは空文字
      name: "ライオン"
      tmpl: 2
      condition: "homework"
      photoData: ""         # 写真シールの場合のみ Base64 文字列（絵文字シールは省略）
      photoFrame: "circle"  # 写真シールのフレーム種別: "circle" | "rounded" | "heart"
```

### 7-4. バリデーション

- 授業態度・テスト項目は授業日（月・木）の日付キーにのみ保存
- 消費ポイント: 1以上の整数、残高以下であること
- ファミリーコード: 4文字以上の英数字

## 8. デザイン方針

| 項目 | 方針 |
|------|------|
| レイアウト | スマホファースト、PC対応 |
| カラー | 明るくポップ（子ども向け）、青系をベースに |
| タップ領域 | 大きめ（子どもが操作しやすい） |
| アニメーション | ポイント登録時にキラキラ・スター演出、シール獲得時にフルスクリーンお祝い演出 |
| フォント | 丸みのある親しみやすい日本語フォント |

## 9. 技術スタック

| 項目 | 内容 |
|------|------|
| 形式 | シングルHTMLファイル |
| スタイル | CSS（インライン） |
| ロジック | Vanilla JavaScript |
| データ保存 | Firebase Realtime Database |
| 外部依存 | Firebase SDK（CDN読み込み、compat v10.12.2） |
| オフライン | Firebase SDK がキャッシュで補完 |

### Firebase プロジェクト設定

- プロジェクト名: `ao-point-app`
- Realtime Database ロケーション: `asia-southeast1`
- ルール: ファミリーコードを知っている人のみ読み書き可能
