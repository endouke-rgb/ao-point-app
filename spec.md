## 1. アプリ概要

| 項目 | 内容 |
|------|------|
| アプリ名 | アオのポイント管理アプリ |
| 対象ユーザー | アオ・保護者 |
| 形式 | シングルHTMLファイル（ブラウザで動作） |
| 保存先 | **Firebase Realtime Database（クラウド）＋ localStorage（オフライン補完）** |
| 目的 | 公文学習の習慣化をポイント制とシール帳で楽しく後押しする |

---

## 2. ユーザー・モード

**モード分けなし。** 全員が同じ画面でポイントの登録・確認・消費を行う。

## 4. 画面・機能一覧

### 4-1. ホーム画面（カレンダー）

- 月単位のカレンダーを表示
- 月の移動ボタン（前月・翌月）
- 授業日（月・木）をカラーでハイライト
- ポイントが記録された日にバッジ（獲得pt数）を表示
- 日付をタップ → ポイント登録パネル（モーダル）を開く
- 画面上部に累積ポイントを常時表示

### 4-2. ポイント登録パネル（モーダル）

```
選択中: ○月○日（曜日）

【宿題（前日までに完了）】
  □ 算数  +1pt
  □ 国語  +1pt
  □ 英語  +1pt

【公文 授業態度】※月・木のみ表示
  □ 集中してやりたくないと言わずに受けた  +1pt

【テスト100点】※月・木のみ表示
  □ 算数  +1pt
  □ 国語  +1pt
  □ 英語  +1pt

【塾 授業態度】※金曜日のみ表示  ★新規追加
  □ 集中して最後まで授業を受けられた  +1pt

【野菜チャレンジ】※毎日表示  ★新規追加
  □ 朝ごはんで嫌いな野菜をたべた  +0.5pt
  □ 夕ご飯で嫌いな野菜をたべた  +0.5pt

【ダンスレッスン】※日曜日のみ表示  ★新規追加
  □ 集中してダンスができた  +0.5pt

【合唱の練習】※土・日のみ表示  ★新規追加
  □ 集中して最後まで合唱の練習を頑張れた  +0.5pt

【学校のテスト】※月〜金のみ表示  ★新規追加
  □ 学校のテストで100点を取った  +1pt

獲得予定: ○pt
[登録する]ボタン
```

- 月・木以外では「公文 授業態度」「テスト100点」セクションを非表示
- 登録済みの日を再度開いた場合、既存の状態を表示（上書き保存）
- 二重加算なし（上書き方式）
- 保存時にシール獲得条件を判定し、条件を満たした場合はシール獲得フローへ遷移

### 4-3. ポイント消費画面

```
現在のポイント: ○○pt

消費するポイント数: [____]
メモ（何に使ったか）: [________]

[ポイントを使う]ボタン
```

- 残高不足の場合はエラーメッセージを表示
- 消費履歴に記録（日付・消費pt・メモ）

### 4-4. シール帳画面

ヘッダー内に2つのタブを表示し切り替えられる。

```
[🏷️ ゲットしたシール]  [✨ スペシャル図鑑]
```

#### タブ①：ゲットしたシール（デフォルト）

- 獲得したシールを一覧表示（4列グリッド、紙のテクスチャ背景）
- 各シールに絵文字・名前・獲得日・獲得条件を表示
- シールは古い順に並ぶ
- シールがない場合は「まだシールがないよ」メッセージを表示
- シールをタップ → シール拡大モーダルを表示

#### タブ②：スペシャル図鑑 ★新規追加

- スペシャルシール全90種類を4列グリッドで一覧表示
- 上部に「X / 90 種類コレクト！（○%）」とプログレスバーを表示
- **ゲット済みシール**
  - カラー表示（フル表示）
  - ✨バッジをシール右上に表示
  - キャラクター名とゲット日（MM/DDゲット）を表示
  - タップ → シール拡大モーダルを表示
- **未ゲットシール**
  - グレーアウト（grayscale + opacity 0.28）
  - 🔒オーバーレイを表示
  - 名前欄は「？？？」と表示

```
✨ 3 / 90 種類コレクト！（3%）
████░░░░░░░░░░░░░░░░░░░░░░░  3%

[ゲット済み]     [未ゲット]      [未ゲット]      [未ゲット]
 カラー+✨        🔒グレー        🔒グレー        🔒グレー
 キャラ名          ？？？          ？？？          ？？？
 5/7ゲット
```

#### シール拡大モーダル

```
[×]ボタン（右上）

シール（大サイズ表示）
名前
獲得日
獲得条件
```

- ダークオーバーレイ（背景タップで閉じる）
- 絵文字シールはフォントサイズ大きめで表示
- 写真シールは元のフレーム形状（丸・角丸・ハート）を維持して拡大表示
- スペシャルシールは図鑑タブからもタップで開ける
- 表示アニメーション：スケールイン（scale 0.5 → 1.0）

### 4-5. 履歴画面

- ポイント獲得履歴（日付・項目・獲得pt）
- ポイント消費履歴（日付・消費pt・メモ）
- 累積獲得 / 累積消費 / 残高 のサマリー
- 各レコードに「削除」ボタンを追加
  - 削除前に確認ダイアログを表示（「この記録を削除しますか？」）
  - 獲得記録を削除した場合 → カレンダーのバッジも同時に消える
  - 消費記録を削除した場合 → 残高が即時回復する
  - 削除後はヘッダーの残高ポイントをリアルタイム更新

## 5. シール帳機能 ★新規追加

### 5-1. シール獲得条件

| 条件 | トリガー |
|------|---------|
| 宿題コンプリート | 1回の登録で算数・国語・英語の3科目すべての宿題にチェックが入った時 |
| テスト100点 | 授業日（月・木）に1科目以上のテスト100点にチェックが入った時 |
| 野菜チャレンジ達成 ★新規追加 | 同じ日の朝ごはん・夕ご飯の両方で野菜チャレンジにチェックが入った時 |

- 再編集時は前回未達成→今回達成の場合のみシールを付与（二重付与なし）
- 1回の保存で複数条件を同時に満たした場合は順番にシール獲得フローを繰り返し実行

### 5-2. シール獲得フロー

```
条件達成
  ↓
派手な演出（全画面お祝いアニメーション）
  ↓
ミニゲーム選択画面
  ├─ [スロット] ボタン
  ├─ [ガチャ] ボタン
  └─ [ビンゴ] ボタン  ★新規追加（スクラッチくじを廃止し追加）
  ↓
ミニゲーム実行
  ├─ 【当たり】
  │    スペシャルシール選択（BONBONドロップシールキャラクター 10種から1つ選択）
  │      ↓
  │    Firebase に保存 → シール帳画面へ遷移
  └─ 【ハズレ】
       通常シール選択モーダル（絵文字シール / 写真シール）
         ↓
       Firebase に保存 → シール帳画面へ遷移
```

- スキップ可能（「あとで選ぶ」ボタン）※ハズレ時の通常シール選択のみ

### 5-3. お祝いアニメーション

- 全画面ダークオーバーレイ（z-index: 300）
- 放射状グラデーション回転アニメーション
- 絵文字がランダムに降るエモジレイン演出
- バイブレーション（Web Vibration API）

### 5-4. ミニゲーム選択画面

- お祝いアニメーション後に表示
- 「スロット」「ガチャ」「ビンゴ」の3択ボタンを大きく表示 ★変更
- 各ボタンにゲーム内容を示すアイコン・説明文を付与
- 当たればスペシャルシールが手に入ることを明示

```
🎰 スロット         🎱 ガチャ             🎯 ビンゴ
  リールを回して       カプセルを             ビンゴになれば
  3つそろえよう！      まわしてゲット！        スペシャルゲット！
```

### 5-5. スロットゲーム ★新規追加

- 3リール × 5シンボル構成
- シンボル一覧：🍬（キャンディ）/ 🍭（ぺろぺろ）/ 🍫（チョコ）/ 🍡（だんご）/ 🍩（ドーナツ）
- **演出**
  - [回す！] ボタンタップでリール順次停止アニメーション（左→中→右、各0.8秒）
  - 停止ごとに効果音相当のバイブレーション
  - 3つ揃い判定後に結果表示
- **当たり条件**：3リールのシンボルがすべて一致
- **当たり確率**：約33%（内部でランダム生成、子ども向けに高め設定）
- **当たり演出**：画面フラッシュ + 「やったー！スペシャルシール！」テキスト + エモジレイン
- **ハズレ演出**：「ざんねん…でも通常シールをもらえるよ！」テキスト

### 5-6. ビンゴゲーム ★新規追加（スクラッチくじの代替）

3×3マス（中央FREE）のビンゴカードで遊ぶゲーム。

#### カード構成

```
┌────┬────┬────┐
│ 🍬 │ 🍭 │ 🍫 │
├────┼────┼────┤
│ 🍡 │ ⭐ │ 🍩 │  ← 中央は FREE（⭐、最初から確定マーク済み）
├────┼────┼────┤
│ 🧁 │ 🍰 │ 🍮 │
└────┴────┴────┘
```

- **シンボル**：🍬 🍭 🍫 🍡 🍩 🧁 🍰 🍮（8種）＋ 中央FREE（⭐）
- カードはゲーム開始時にランダム配置で生成

#### 操作フロー

```
① ビンゴカードが表示される（中央⭐は最初からマーク済み）
    ↓
② [つぎ！] ボタンをタップ → 1つシンボルが抽選され発表
    ↓  カードの該当マスがハイライト（ぴかっと光る）
③ ②を繰り返す（最大7回コール可能）
    ↓
  ビンゴ成立（縦・横・斜めいずれか1ライン揃い）→ 当たり
  7回コールしてもビンゴなし                    → ハズレ
```

#### ゲーム仕様

- **ビンゴ条件**：縦3列・横3列・斜め2本の計8ライン中、1本以上完成
- **最大コール数**：7回（中央FREE含め最大8マスマーク可能）
- **当たり確率**：完全ランダム（カード配置・抽選順ともに操作なし）
  - カードの絵柄配置はゲーム開始時にランダムシャッフル
  - コール順も毎回ランダムシャッフル
  - 7コール以内に自然にビンゴが成立すれば当たり
- **成立ライン演出**：ビンゴ成立ライン上のマスが点滅 → フラッシュ + 「🎯 BINGO!!」テキスト + エモジレイン + バイブレーション
- **ハズレ演出**：「ざんねん… でも通常シールをもらえるよ！🎁」

#### UIポイント

- マーク済みのマスは背景色変化（グレー → ゴールド）でわかりやすく表示
- コール済みシンボルのリストを画面下部に表示（「でたもの：🍬 🍭 …」）
- ビンゴ成立ラインは金色ハイライトで強調

### 5-7. ガチャゲーム ★新規追加

カプセルトイマシン（ガチャガチャ）をモチーフにしたゲーム。

#### 画面構成

```
🎱 ガチャガチャ

  ┌──────────────┐
  │   🎰マシン絵  │   ← カプセルトイマシン（CSS絵文字で表現）
  │   出口: [ ]   │   ← カプセル出口
  └──────────────┘

  [まわす！🎱] ボタン

  ※当たりが出たら✨スペシャルシールゲット！
```

#### 操作フロー

```
① [まわす！] ボタンをタップ
    ↓
② ハンドル回転アニメーション（くるくる、0.8秒）
    ↓
③ カプセルが出口からポトッと落下（落下アニメーション、0.6秒）
    ↓
④ カプセルがバウンド・揺れる（0.4秒）
    ↓
⑤ カプセルがパカッと開く（上下2分割アニメーション、0.5秒）
    ↓
⑥ 当たり or ハズレ 結果表示
```

#### ゲーム仕様

- **カプセルの色**：5色からランダム（赤・青・黄・緑・ピンク）
- **当たり確率**：約33%（スロット・ビンゴと同一）
- **内部判定**：ボタンタップ時に事前決定（演出開始前）
- **当たり演出**：カプセルが開いた瞬間にフラッシュ + 「✨スペシャルシール！」テキスト + エモジレイン + バイブレーション
- **ハズレ演出**：カプセルが開いた瞬間に「ざんねん…でも通常シールをもらえるよ！🎁」テキスト

### 5-8. スペシャルシール（当たり時） ★新規追加

当たった場合のみ選択できるBONBONドロップシールキャラクターモチーフの特別シール。

#### スペシャルシール 10種

| # | 絵文字 | 名前 | カラーテーマ |
|---|--------|------|-------------|
| 1 | 💝 | キャンディハート | ピンク × レッド |
| 2 | 🍋 | レモンドロップ | イエロー × グリーン |
| 3 | 🍒 | チェリーボム | レッド × ダークレッド |
| 4 | 🍇 | グレープソーダ | パープル × ブルー |
| 5 | 🍓 | ストロベリークリーム | レッド × ホワイト |
| 6 | 🫐 | ブルーベリーポップ | ブルー × パープル |
| 7 | 🍊 | オレンジスパーク | オレンジ × イエロー |
| 8 | 🌿 | ミントアイス | グリーン × ホワイト |
| 9 | ⭐ | チョコレートスター | ブラウン × ゴールド |
| 10 | 🌈 | レインボーキャンディ | 虹色グラデーション |

- 10種すべてを一覧表示（4列グリッド）
- 各シールはアニメーション付きで登場（順番にスケールイン）
- 1つ選択後に [このシールにする！] ボタンで確定
- スペシャルシールはシール帳で特別フレーム（金縁）で表示
- データ保存時に `isSpecial: true` フラグを付与

### 5-9. 通常シールデザイン（ハズレ時）

- 絵文字ベースのシール（外部API不要）
- 5種のビジュアルテンプレート（円形・角丸正方形、グラデーション色違い）
- 約75種のシールデータベース（キーワード検索対応）
- 検索ワード例：「動物」「食べ物」「宇宙」「お姫様」など

### 5-10. 写真シール機能

- シール選択モーダルに「写真シール」タブを追加
- **写真入力方法**
  - ファイル選択（端末のカメラロール・ファイルから選択）
  - カメラ直接撮影（`capture="environment"` または `capture="user"` 対応端末のみ）
- **シール枠（フレーム）**：3種類から選択
  - 丸型（circle）
  - 角丸正方形（rounded）
  - ハート型（heart） ※SVG clip-path で実装
- **プレビュー**: 選択した写真 + フレームをリアルタイムでCanvas描画してプレビュー表示
- **保存形式**: Canvas を `toDataURL('image/jpeg', 0.6)` で Base64 化し Firebase Realtime Database に保存
  - 1枚あたり概算サイズ: 58×58px JPEG ≒ 2〜5KB（Firebase 無料枠内で十分）
- **シール帳表示**: `emoji` フィールドが空で `photoData` フィールドがある場合は `<img>` タグで表示、フレーム形状を CSS で適用
- **バリデーション**
  - 対応形式: image/*（JPEG/PNG/GIF/WEBP）
  - ファイルサイズ上限: 10MB（超えた場合はエラーメッセージ）

## 7. データ設計

### 7-1. 保存先

- **Firebase Realtime Database** を使用（無料枠：1GB保存 / 10GB転送／月）
- データはリアルタイムに全端末へ自動反映される
- オフライン時はFirebase SDKがローカルキャッシュで補完し、復帰後に自動同期

### 7-2. 同期の仕組み

```
親のスマホ         Firebase              子のタブレット
   │   ─── 書き込み ──→   DB    ─── リアルタイム通知 ──→  │
   │   ←── 更新受信 ───   DB    ←─────────────────────  │
```

- 同一の「ファミリーコード（任意の文字列）」を両端末で入力することで同じデータを共有する
- ファミリーコードは初回起動時に設定画面で入力

- ファミリーコード: 4文字以上の英数字
